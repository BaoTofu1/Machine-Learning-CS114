# Bank Marketing Campaign Prediction Web Demo

Streamlit demo for the fixed Bank Marketing machine learning project.

The app predicts whether a customer is likely to subscribe to a term deposit. It is designed for classroom demonstration of the complete workflow: EDA, preprocessing, model training, model comparison, and interactive prediction.

## What Was Fixed

- The app now finds `processed_bank_data.csv` by file location, so it can be launched from the project root or from the `Web Demo` folder.
- The processed dataset is rebuilt from the raw UCI zip using relative paths.
- `is_contacted_before` is included in the processed dataset.
- Exact duplicate raw rows are removed during preprocessing.
- The realistic model scenario excludes `duration`, because call duration is only known after the call.
- Documentation no longer references missing quickstart scripts.

## App Pages

- Home: project overview and dataset statistics.
- Data Exploration: processed data preview, missing values, target distribution, numerical distributions, and correlation.
- Model Training: train one selected algorithm and view metrics/curves/feature importance.
- Make Predictions: enter a customer profile and view predicted subscription probability.
- Model Comparison: compare Logistic Regression, XGBoost, Random Forest, and LightGBM.

## Modeling Notes

Two scenarios are intentionally kept:

- Scenario 1, with `duration`: benchmark only. This scenario has leakage because `duration` is not known before the call.
- Scenario 2, without `duration`: realistic scenario for campaign targeting before calling a customer.

Because the target class is imbalanced, do not rely on accuracy alone. Prefer ROC-AUC, PR-AUC, F1 for class 1, recall, precision, and Balanced Accuracy.
